# AI Agent 赛道日报 2026-05-13

## 摘要
今日赛道整体活跃：阿里 QwenPaw 发布 v1.1.7-beta.1，引入自动记忆管理与异步执行能力；OpenClaw 密集迭代，单日推送三个 Beta 版本，新增基于发送者身份的工具访问策略与网关参数扩展；Manus 上线自定义浏览器绑定功能，持续完善自动化工作流生态。

## 阿里 QwenPaw

- **v1.1.7-beta.1**（2026-05-12）发布，核心新特性：
  - 自动记忆管理（auto-memory management）：Agent 可自主维护跨会话记忆
  - 异步执行支持（async execution support）
  - 聊天界面支持多附件同时上传
  - Skill 系统架构重构
  - 钉钉引用消息（DingTalk quoted message）解析优化
  - 浏览器操作批量执行优化（browser_use batch actions）
  - 飞书语音气泡支持；新增印尼语（Indonesian）语言包
  - 来源：https://github.com/agentscope-ai/QwenPaw/releases

## OpenClaw

- **v2026.5.12-beta.4**（2026-05-13 05:17 UTC）发布，当日第三个 Beta 版本，主要变更：
  - 网关层新增 max_completion_tokens / max_tokens 参数兼容，覆盖 OpenAI-compatible 及 Azure 服务
  - 新增基于发送者身份的工具访问策略（per-sender tool policies），支持 per-agent 级别覆盖
  - 本地模型服务支持按需启动（provider-level on-demand startup）
  - pnpm 11 工作区升级；TypeScript 及 lint 配置严格化
  - Slack 链接/媒体展开控制；Telegram HTML 标签保留；iMessage 状态过滤
  - 来源：https://github.com/openclaw/openclaw/releases
- 社区文章《为什么我把一个 Agent 拆成了六个》（2026-05-12，53AI）：从上下文工程角度探讨多 Agent 任务拆分实践，核心论点为上下文膨胀是拆分的主要驱动力。来源：https://www.53ai.com/news/Openclaw/

## Manus

- **Introducing Preferred Browser**（2026-05-12，Academy）：支持为自动化流程绑定指定浏览器，提升端到端工作流执行的可控性与一致性。来源：https://manus.im/blog
