# AI Agent 赛道日报 2026-07-02

## 摘要

今日赛道动态集中在开源项目版本更新：QwenPaw 发布 v2.0.0-beta.2，引入滚动上下文管理器、cron 更新命令等；OpenClaw 发布 v2026.7.1-beta.1，新增 GPT-5.6 支持、Telegram Codex 工作流等。Stanford HAI 发布新研究《AI Coding Agents Fail at Teamwork》，揭示 AI 编码智能体协作能力缺陷。

## 重大信号

- [QwenPaw] QwenPaw v2.0.0-beta.2 发布：架构迁移至 AgentScope 2.0，引入滚动上下文管理器、cron 更新命令、DashScope 模型列表更新等。
- [OpenClaw] OpenClaw v2026.7.1-beta.1 发布：新增 GPT-5.6 支持、Telegram Codex 工作流、事件驱动 cron 等。
- [Stanford HAI] 新研究《AI Coding Agents Fail at Teamwork》：两个模型协作比单个更差，暴露 AI 协作能力关键缺口。

## 阿里 QwenPaw

- 发布 v2.0.0-beta.2（2026-07-02）：
  - 新增滚动上下文管理器（scroll context manager），支持持久化历史与召回 REPL
  - 新增 cron 更新命令（cron update），支持修改已有定时任务
  - 更新 DashScope 内置模型列表至最新 Qwen3.7 和 DeepSeek V4 系列
  - 修复多项问题：记忆系统提示词优化、工具启用/禁用配置感知、Langfuse 追踪分组恢复、桌面截图功能修复等
  - 文档更新：上下文管理、安全沙箱、Terminal UI 等
  - 来源：https://github.com/agentscope-ai/QwenPaw/releases/tag/v2.0.0-beta.2

## 开源 OpenClaw

- 发布 v2026.7.1-beta.1（2026-07-02）：
  - 新增 GPT-5.6 模型家族支持
  - 新增外部 harness 附加命令（openclaw attach），可附加到已有 Gateway 会话
  - Telegram 支持 Codex 工作流：/login 配对、引导 Codex 运行、恢复最终回复
  - 新增事件驱动 cron（on-exit 调度类型），当监控命令退出时唤醒 agent
  - 原生应用刷新：iOS 采用 iOS 26 视觉系统，导航、设置、Chat/Talk 流程改进
  - iMessage 原生投票创建/读取/投票功能
  - 更安全的会话能力配置文件（capability profiles）
  - 来源：https://github.com/openclaw/openclaw/releases/tag/v2026.7.1-beta.1

## Stanford HAI

- 新研究《AI Coding Agents Fail at Teamwork》（2026-06-01）：
  - 两个 AI 编码模型协作时性能比单个更差，暴露 AI 在团队协作中的关键能力缺口
  - 来源：https://hai.stanford.edu/news/ai-coding-agents-fail-teamwork
